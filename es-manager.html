How does this thing work?

Our golang code generates external secret yaml files by interacting with the vault api and creates files with keys present which are to be created as secrets in kubernetes

Okay but why do we need this?

One problem with vault is that we have to first enter the secrets in the vault UI and then create external secrets again for the same secrets. This tool does it for you.

How can I use external-secrets-manager?

It's open source! You just need to use our helm install commands to set it up on your kubernetes environment in a working vault external secret setup. The rest is handled for you.
